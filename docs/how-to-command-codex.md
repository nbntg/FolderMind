# FolderMind 分步骤开发指令手册

这个文档的目的不是写产品规格，而是教你以后怎么分批命令 Codex。每次只让 Codex 做一组明确的小任务，做完后先测试，再进入下一步。

## 使用原则

1. 每次只做一个阶段，不要一次塞太多需求。
2. 每次开始前先说清楚“只分析，不改代码”还是“直接改代码”。
3. 涉及文件整理、删除、撤销、历史记录时，必须要求 Codex 写测试。
4. 涉及 UI 时，必须要求 Codex 构建并检查界面，不要只改代码。
5. 如果出现卡死、乱码、状态丢失、数据没保存，先让 Codex 定位原因，再让它修改。

## 推荐总路线

建议按这个顺序推进：

1. 执行安全与撤销
2. JSON 校验与兼容
3. 冲突处理
4. 历史记录
5. AI 对话窗口
6. 设置与配置持久化
7. 大文件夹体验
8. 打包 EXE

不要跳过前面的基础能力直接做打包。否则后面发现数据丢失、撤销不完整、路径错误，会更难查。

## 阶段 1：执行安全与撤销

目标：确保执行整理之前能预览，执行之后能完整撤销。

你可以这样命令：

```text
从文，这次只做执行安全和撤销，不要改其他功能。

要求：
1. 执行前必须生成可撤销记录。
2. 撤销时要恢复移动过的文件。
3. 撤销时要删除执行中新建的空文件夹。
4. 如果文件夹不为空，不能强删，要记录为跳过并说明原因。
5. 撤销不能一直卡在“撤销中”。
6. 给这些逻辑补测试。

做完后运行后端测试，并告诉我测试结果。
```

验收重点：

- 移动文件后能撤回原位。
- 新建的空目录会被删掉。
- 非空目录不会被误删。
- 按钮状态不会卡住。

## 阶段 2：JSON 校验与兼容

目标：无论 AI 返回 213 那种格式，还是 231 那种格式，程序都不会卡死。不能执行的地方要明确告诉用户原因。

你可以这样命令：

```text
从文，这次只做 JSON 导入、解析、校验和兼容，不要改 UI 大布局。

我会给你两个 JSON 示例文件。你先比较它们的结构差异，然后修改解析器：
1. 支持 FolderMind 标准 actions 格式。
2. 尽量兼容树状 JSON 或分类 JSON。
3. 不兼容时不能卡死，要显示“无法识别格式”和具体原因。
4. 导入 JSON 后，预览和执行都应该可用，除非校验失败。
5. 给解析器和预览逻辑补测试。

做完后用我给的两个文件分别验证。
```

验收重点：

- 坏 JSON 不会卡死。
- 非标准 JSON 会给出明确错误。
- 可转换的 JSON 会被转换成 actions。
- 预览不会一直 Ready 或闪烁。

## 阶段 3：冲突处理

目标：用户不只是看到冲突，还能处理冲突。

你可以这样命令：

```text
从文，这次只做冲突处理功能。

要求：
1. 预览时列出缺失、冲突、错误。
2. 冲突项可以点击查看详情。
3. 对每个冲突提供处理方式：跳过、自动重命名、覆盖、手动改路径。
4. 默认不能覆盖用户文件。
5. 执行时按用户选择处理。
6. 历史记录里保存最终处理结果。

先给我一个简短设计方案，我确认后再改代码。
```

验收重点：

- 冲突不是只显示数字。
- 能看到具体哪个文件冲突。
- 默认策略保守，不误删不覆盖。

## 阶段 4：历史记录

目标：历史记录真正有用，能保存、读取、查看、删除。

你可以这样命令：

```text
从文，这次只做历史记录。

要求：
1. 历史记录保存在程序目录下的 history 文件夹。
2. 重新打开软件必须能读取以前的历史。
3. 同一次整理流程中，扫描、预览、执行要合并成一条记录。
4. 记录里要显示成功、跳过、错误、冲突、缺失。
5. 点击记录后打开详情页。
6. 支持删除单条记录，删除前弹确认框。
7. 支持清空历史，清空前弹确认框。

做完后请你关闭再重新打开程序验证一次。
```

验收重点：

- 关闭软件后历史不消失。
- 删除单条后重新打开不会复活。
- 一次流程不是三条记录。

## 阶段 5：AI 对话窗口

目标：AI 不是一次性发送，而是一个可持续对话的助手。

你可以这样命令：

```text
从文，这次只做 AI 对话窗口，不要改执行器。

要求：
1. 点击“发送给 AI”后出现悬浮球。
2. 悬浮球切换页面后仍然存在。
3. 点击悬浮球打开聊天窗口。
4. 第一条消息显示本次发送的提示词和文件列表。
5. 支持多轮对话输入。
6. 支持中断当前请求。
7. 支持复制当前对话内容。
8. AI 返回 JSON 后，可以一键导入到 JSON 区。
9. 聊天内容可以选择和复制，但不能直接编辑。

做完后请重点检查：切换页面、关闭窗口、重新打开窗口、超时中断。
```

验收重点：

- 不是只能发一次。
- 超时可以中断。
- 返回内容不会丢。
- 对话能复制。

## 阶段 6：设置与配置持久化

目标：用户填过的设置不会因为切页面或重启而消失。

你可以这样命令：

```text
从文，这次只做设置页和配置持久化。

要求：
1. Anthropic、OpenAI、Custom 三个服务商分别保存 endpoint、模型、API Key。
2. 切换服务商时显示各自的配置，不要共用同一份输入框数据。
3. API Key 输入框右侧加眼睛按钮，支持显示和隐藏。
4. 测试连接要显示成功或失败，以及失败原因。
5. 关闭软件再打开，配置仍然存在。
6. 增加恢复默认设置按钮，点击前必须确认。

做完后请你重启程序验证配置是否还在。
```

验收重点：

- 切换页面不丢配置。
- 重启不丢配置。
- 不同服务商配置互不污染。

## 阶段 7：大文件夹体验

目标：扫描大文件夹时不会卡死，用户知道程序在做什么。

你可以这样命令：

```text
从文，这次只优化大文件夹体验。

要求：
1. 扫描时显示进度和文件数量。
2. 扫描过程中可以取消。
3. 文件列表很长时，只让文件结构区域滚动，不让整个页面乱滚。
4. 文件内容过大时，“复制内容”自动变成“导出文件”。
5. UI 不能因为长文件名被撑爆，长文件名要换行或截断。

做完后用一个包含大量文件的文件夹测试。
```

验收重点：

- 长文件名不撑坏页面。
- 大目录扫描有反馈。
- 页面底部按钮始终可见。

## 阶段 8：打包 EXE

目标：做成双击可运行的 EXE，不走安装程序。

你可以这样命令：

```text
从文，现在开始做 EXE 打包方案。

要求：
1. 打包成双击运行的 EXE，不做安装程序。
2. history 文件夹放在 EXE 同级目录。
3. config 文件夹也放在 EXE 同级目录。
4. 重启 EXE 后历史记录和配置都能读取。
5. 给我一个 build 脚本。
6. 告诉我打包后的目录结构。

先不要急着打包，先给我方案和风险点。
```

验收重点：

- 换电脑或换目录后仍能运行。
- 历史和配置跟着 EXE 目录走。
- 不依赖开发环境启动。

## 每次让 Codex 修改时的固定模板

你可以直接复制这段：

```text
从文，这次只处理【这里写功能名】。

范围：
1. 只改和这个功能相关的文件。
2. 不要顺手重构其他模块。
3. 不要改变已经能正常工作的功能。

要求：
1. 【写具体要求 1】
2. 【写具体要求 2】
3. 【写具体要求 3】

验证：
1. 需要补测试。
2. 需要运行后端测试。
3. 如果改了前端，需要运行前端 build。
4. 如果是 UI 问题，需要说明你检查了哪些界面状态。

完成后告诉我：
1. 改了哪些地方。
2. 怎么验证的。
3. 还有什么风险。
```

## 遇到 Bug 时的固定模板

```text
从文，先不要改代码，先定位原因。

现象：
【这里描述问题】

复现步骤：
1. 【第一步】
2. 【第二步】
3. 【第三步】

我希望你先回答：
1. 这个问题最可能发生在哪一层。
2. 你准备检查哪些文件。
3. 需要我提供什么截图或文件。

等我确认后你再改代码。
```

## 我建议你优先让我做的三件事

如果你想继续推进，建议按下面三个任务走：

1. JSON 校验与兼容：这是最容易造成卡死和执行失败的地方。
2. AI 对话窗口：这是 FolderMind 和普通文件整理工具最大的差异点。
3. 历史记录详情与删除：这是用户信任软件的重要基础。

这三个做完之后，再考虑冲突手动处理和 EXE 打包。
